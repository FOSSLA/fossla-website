# Contributing to the FOSSLA Blog

We welcome contributions to the FOSSLA Blog! Whether you want to share your knowledge, write about open source, or document community events, your input is valuable.

## How to Contribute

1. **Fork the Repository**
   - Click the "Fork" button on GitHub to create your own copy of the repository.

2. **Create a New Blog Post**
   - Navigate to `src/content/blog/` in your fork.
   - Add a new file with a descriptive name, e.g., `my-awesome-post.md`.
   - Use Markdown (`.md`) or plain text (`.txt`).
   - Start your post with a title and (optionally) metadata like author and date.

3. **Write Your Content**
   - Share your story, tutorial, event recap, or opinion.
   - Use clear language and add images or code snippets if needed.

4. **Commit and Push**
   - Commit your changes with a meaningful message.
   - Push to your forked repository.

5. **Open a Pull Request**
   - Go to the original repository on GitHub.
   - Click "New Pull Request" and select your branch.
   - Describe your changes and submit the PR.

## Blog Post Template

```
---
title: "Your Blog Title"
author: Your Name
date: YYYY-MM-DD
---

Your content goes here...
```

## Tips
- Keep posts relevant to FOSS, technology, or the Ladakh community.
- Proofread your post before submitting.
- Be respectful and follow our [Code of Conduct](/about#conduct).

Thank you for contributing to the FOSSLA Blog!
